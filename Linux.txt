Linux & Linux Internals: Day 1/15 Syllabus
1. Linux Architecture & Internals (The Core)
• The Layers: Hardware -> Kernel -> Shell -> User Space Applications.

• Kernel Space vs User Space:

  • Kernel Space: Protected memory area where the core OS, memory management, and hardware drivers run. Full hardware access.

  • User Space: Restricted memory area where user applications (Python scripts, Web servers, `ls` command) run.

• System Calls (Syscalls): How User Space applications request privileges or hardware access from the Kernel (e.g., `open()`, `read()`, `write()`, `fork()`).

2. File System Hierarchy Standard (FHS) Internals
• `/etc`: System-wide configuration files (e.g., network settings, user databases).

• `/var/log`: Centralized system and application logs (Critical for SRE troubleshooting).

• `/proc`: A virtual/pseudo filesystem in RAM (not on disk) providing real-time kernel, hardware, and running process metadata.

• `/bin` & `/sbin`: Essential user and system administration command binaries.

3. Kernel Inspection & Navigation Commands
• `ls -ltr`: Lists files sorted by modification time in reverse (SRE favorite for finding recent logs).

• `cat /proc/cpuinfo`: Inspects CPU architecture directly from the Kernel.

• `cat /proc/meminfo`: Inspects real-time memory usage and availability.

• `man <command>`: Accesses the official on-system reference manual.

Practical Tasks for Tomorrow Morning:
1. Navigate to `/var/log` and run `ls -ltr` to identify the most recently updated log file.

2. Run `cat /proc/meminfo` and inspect `MemTotal` vs `MemAvailable`.

3. Open `man grep` to study system call level integration and filtering options.
