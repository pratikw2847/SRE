3. FILE SYSTEM & DIRECTORY STRUCTURE
==================================================

- Linux File System Hierarchy
OS stores and manages data on disk using partitions called file system
File system consits of files directories and its related permissions
 
How to check linux file system
lsblk -f
df -Th
cat /etc/fstab

types of file system
XFS and EXT4

- Important Directories
  - /root - linux file system starts with root directory which is denoted by "/" from which all other files and directories are branched. Top level directory and home directory for root user
  - /bin - contains essential user binaries such as commands like cp ls 
  - /etc - contains all system configuration files - files related user networking dns and all
  - /var - contains all variable data such as logs are stored
  - /home - contains personal directories of all users
  - /tmp - temperory files created by all users and system
  - /proc - A virtual or dyanamic directory that only exist in memory. It does not use disk space.
  - /sys - 
  - /dev - contains all files which represent and provide access to hardware devices such as harddrives 
  - /boot - Holds file related to system bootup including kernal, inital ram storage and all
  - /lib /lib64 - These directories contains essential shared libraries and kernal modules that are needed to boot the system
  - /mnt - this is traditional mount point where system administrors can mount temp filesystems 
  - /opt - Intented for the installation of add-on application packages. software packages that are not part of default installation.
  - /run - A temp filesystems that stores transiant state files like process ids and lock files since it is created at every boot.
  
- Absolute vs Relative Paths
Absoulte Path - An absolute path starts from the root directory (/) and gives the complete location of a file or folder.
Relative Path - A relative path is based on your current working directory.

7. PROCESS MANAGEMENT
==================================================

- What is a Process 


- Process Lifecycle
- PID
- Parent & Child Process
Parent: The process that creates a new process
Child: The new process created by the parent using the fork() system call.

- Foreground & Background Process
Foreground: A process that takes control of your terminal. You cannot run other commands until it finishes
Background: A process that runs independently of the terminal interface. You can send a process to the background by adding an & at the end

- Daemon Process
Definition: A background process that runs continuously from system boot until shutdown. They are not attached to any user session.

Identification: Their names usually end with a 'd' (e.g., sshd, httpd, systemd).

- Zombie Process
The Cause: It occurs when a Child finishes, but the Parent fails to call wait() to read the child's exit status.

Identification: In top, they appear as "zombie" or "defunct". In ps, they have a 'Z' status.

Impact: They don't consume CPU or RAM, but they occupy a PID (Process ID) slot. If the PID limit is reached, the system cannot start new processes.

- Orphan Process
Definition: A child process that is still running, but its Parent has terminated or been killed.

The Solution: Linux handles this gracefully. The init process (PID 1) or systemd "adopts" the orphan process and becomes its new parent.
