1. Python Dictionaries (`dict`)
• What is a Dictionary? A collection of Key-Value pairs. In SRE, lists are great for multiple items (like a list of IPs), but Dictionaries are crucial for storing detailed metadata about a single object (like server details, cloud configuration, or user details).


• Syntax: Enclosed in curly braces `{}`. KEYS must be strings, VALUES can be any data type.


🛑 Loop Control Statements: `break` and `continue`

• `break`: Terminates the loop completely when a specific condition is met. (Used to stop looking once you find a critical error).

• `continue`: Skips the current iteration and jumps directly to the next item in the loop. (Used to skip non-critical issues or healthy servers).

🐍 Python Day 5/90: Functions & Error Handling (SRE Automation Engineering)
1. Python Functions (`def`)
• What is a Function? A reusable block of code that performs a specific action. Instead of writing the same 10 lines of code every time you want to check a server, you wrap it in a function and call it with one line.

• Syntax: Defined using the `def` keyword, followed by the function name, arguments in `()`, and indentation.

```

def check_status(server_name):

    print(f"Checking infrastructure connectivity for: {server_name}")

# Calling the function

check_status("prod-web-01")

```


🧠 Return Values (`return`)

Functions can send data back to the main script using the `return` statement. Once a function hits `return`, it exits immediately.

```

def calculate_cpu_free(total, used):

    free = total - used

    return free

remaining_cpu = calculate_cpu_free(16, 12)

print(f"Available CPU: {remaining_cpu} GB")

```

2. Robust Automation: Error Handling (`try-except`)
In SRE, automation scripts run against production systems. If a server is down, your script shouldn't crash (`Traceback Error`); it should handle the failure gracefully, log it, and continue checking other systems. This is called Fault Tolerance.

🛠️ The `try-except-finally` Block Syntax

```

try:

    # Code that might crash (e.g., division by zero, missing keys, connection timeout)

    result = 10 / 0

except ZeroDivisionError as e:

    # Code that runs ONLY if a specific crash happens

    print(f"ALERT: Mathematical anomaly detected -> {e}")

except Exception as e:

    # Catch-all generic block for any other unexpected failures

    print(f"CRITICAL: Unexpected infrastructure failure: {e}")

finally:

    # This block ALWAYS runs no matter what (Used to close open connections/files)

    print("Execution segment finished. Cleaning up system states.")

```

```
def parse_device_config(server_profile):
```
