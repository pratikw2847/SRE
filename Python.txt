2. Robust Automation: Error Handling (`try-except`)
In SRE, automation scripts run against production systems. If a server is down, your script shouldn't crash (`Traceback Error`); it should handle the failure gracefully, log it, and continue checking other systems. This is called Fault Tolerance.


🛠️ The `try-except-finally` Block Syntax

```

try:

    # Code that might crash (e.g., division by zero, missing keys, connection timeout)

    result = 10 / 0

except ZeroDivisionError as e:

    # Code that runs ONLY if a specific crash happens

    print(f"ALERT: Mathematical anomaly detected -> {e}")

except Exception as e:

    # Catch-all generic block for any other unexpected failures

    print(f"CRITICAL: Unexpected infrastructure failure: {e}")

finally:

    # This block ALWAYS runs no matter what (Used to close open connections/files)

    print("Execution segment finished. Cleaning up system states.")

1. The Hash-Map Revolution (Python Dictionaries In Depth)
Dictionaries - it contains key value pair where key is unique identifier and value is data for that key.
 


• Purpose: To achieve microsecond speed when processing millions of server log lines.
• Time Complexity (O(1) Lookup):
  - Searching data inside a standard List takes O(n) time (the script slows down when processing millions of items).
  - Dictionaries use an underlying Hash Map architecture, allowing instantaneous data retrieval for any 'Key' in O(1) time.

4. Building Idempotent Automation Scripts
• Purpose: Ensuring that running a script multiple times will never corrupt the production system's state.
• Idempotency Concept: The script should apply changes during the first run, but safely skip execution without crashing on subsequent runs if the desired state is already met.
• SRE Logic Example:
  Running `os.mkdir("prod_logs")` directly will throw a `FileExistsError` and crash the execution if the folder already exists. It is mandatory to check the current system state using `os.path.exists()` first.
• Syntax:
  import os
  if not os.path.exists("prod_logs"):
      os.mkdir("prod_logs")
      print("Folder created successfully.")
  else:
      print("Folder already exists. Skipping creation to keep idempotency.")
