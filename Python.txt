2. Conditional Statements (Making Decisions)
• if: Executes code only if the condition is true.

• elif: Checks another condition if the previous one was false.

• else: Executes if none of the above conditions are true.

• Indentation: Crucial in Python! Always use 4 spaces (or one Tab) for the code block inside `if/else`.

Python Day 3/90: Lists & For Loops (SRE Essentials)
1. Python Lists (Arrays)
• What is a List? A way to store multiple items in a single variable (e.g., list of servers).

• Syntax: `servers = ["prod-web-01", "prod-db-01", "stage-app-01"]`

• Indexing: Accessing elements using numbers (Starts from 0).

  • `print(servers[0])` -> Prints `"prod-web-01"`

  • `print(servers[-1])` -> Prints the last item (`"stage-app-01"`).

• List Methods (Crucial for SRE):

  • `servers.append("prod-lb-01")` -> Adds a new server to the end.

  • `servers.remove("stage-app-01")` -> Deletes an item.

  • `len(servers)` -> Gives the total count of servers.

2. The 'For' Loop (Automation Backbone)
• Why Loops? Instead of checking 100 servers manually, a loop does it in 2 lines of code.

• Syntax:

```

servers = ["web-01", "web-02", "web-03"]

for server in servers:

    print(f"Deploying application to {server}")

```

3. Combining Loops and If-Else (Real SRE Logic)
```

servers = ["prod-web", "stage-db", "prod-api", "test-vm"]

for server in servers:

    if "prod" in server:

        print(f"CRITICAL: {server} belongs to Production!")

    else:

        print(f"INFO: {server} is a non-prod environment.")

```
